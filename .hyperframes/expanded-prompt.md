# Omni-Presence Demo Wrapper - Expanded Production Prompt

## Title + Style Block

Build three standalone HyperFrames clips for Fred's omni-presence agent demo:

- intro: 8 seconds
- tech-explainer: 14 seconds
- outro: 8 seconds

Visual identity is dark premium console editorial:

- Background: #080B10
- Deep panel: #10141C
- Foreground: #F4F1EA
- Muted text: #B9C0CC
- Red-orange accent: #FF4D2E
- Console green: #35D08A
- Line color: #2B3343

Typography:

- Display: Space Grotesk, 800-900
- Technical labels: IBM Plex Mono, 600-700
- Body: Space Grotesk, 350-500

Mood: cinematic title sequence for applied AI builders. The frame should feel like a premium technical artifact, not a product marketing slide.

## Rhythm Declaration

Pattern: hook-hold-proof-resolve.

- Intro: fast visual hook, hold on thesis.
- Tech explainer: slow build with proof object, then three callouts.
- Outro: quiet but decisive closing line.

## Global Rules

- No live-demo audio inside these clips.
- Use caption-driven visuals so Fred can stitch voiceover or live walkthrough audio separately.
- Avoid gradient text, generic blue-purple neon, and heavy terminal clutter.
- Use seeded/explicit motion only, no random values.
- Use the existing static architecture PNG as the proof artifact in the tech explanation clip.

## Scene Beats

### Intro

Concept: the agent begins trapped in a textbox and then the frame opens outward into real channels. Viewers should understand the thesis in one glance before the live walkthrough starts.

Depth layers: dark field, soft red glow, grid trace, small channel pills, hero headline, command line, edge metadata.

Choreography: command types on, textbox outline draws, headline slams softly into place, channel pills cascade outward, punchline locks in at the bottom.

### Tech Explainer

Concept: the live walkthrough proves the workflow; this bridge explains what was actually built. The existing architecture asset becomes a physical proof board with overlays highlighting the control plane, workflow steps, and skill/CLI layer.

Depth layers: dark backing field, asset image, scanning highlight, three callout panels, connector lines, small implementation labels.

Choreography: asset scales up from blur, scan bar travels, callouts draw in sequence, final control-plane statement appears.

### Outro

Concept: after the live demo, land the big idea. Agents should not wait inside one chat UI; they should meet the user through the channels where work already happens.

Depth layers: dark field, faint channel map, five surface nodes, hero punchline, closing credential line.

Choreography: nodes pulse in sequence, textbox collapses into a small prompt, punchline resolves, credential line fades in.

## Recurring Motifs

- Red-orange for the thesis and action.
- Console green for reliable tool execution.
- Hairline borders and monospace metadata for technical credibility.
- Channel nodes flowing outward from a single agent control plane.

## Negative Prompt

No stock hero image, no generic robot/AI icon, no purple-blue gradient soup, no massive terminal wall, no text that requires pausing to read, no live-call transcript in these wrapper clips.
