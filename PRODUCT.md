# Product

## Register

brand

## Users

Applied AI founders, startup engineers, technical recruiters, and AI builders who are evaluating whether Fred can design and ship practical agent workflows across real APIs and tools.

## Product Purpose

This repo supports a hackathon demo of an omni-presence agent workflow. The public artifact should make one idea easy to understand: Codex/Claude Code can act as a control plane for real-world agent skills instead of remaining a coding assistant trapped in one chat surface.

## Brand Personality

Precise, capable, and frontier-practical. The demo should feel engineered and credible, with enough visual confidence to travel on X, LinkedIn, and Reddit.

## Anti-references

Avoid AI-slop aesthetics: glowing glass panels, fake holograms, generic robot imagery, decorative cloud icon clutter, unreadable microtext, one-note purple/blue gradients, and diagrams that blend user workflow with implementation detail.

## Design Principles

- Show the live user journey before explaining the stack.
- Separate what the user experiences from what powers it.
- Make human approval visible as a first-class design choice.
- Favor exact labels and honest primitives over vague agent magic.
- Make the core thesis memorable: do not let your agent sit in a textbox.

## Accessibility & Inclusion

Use high-contrast text, large labels suitable for mobile video viewing, reduced visual noise, and clear information grouping. Diagrams should remain legible when compressed in social feeds.
